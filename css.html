1. HTML : L'Ossature

Le HTML (HyperText Markup Language) définit le sens et la structure de votre contenu. Il ne s'occupe pas de l'esthétique.

La Notion de Parent et Enfant

Le HTML est un arbre. Une balise imbriquée dans une autre est un enfant (child), et celle qui l'entoure est le parent. C'est crucial pour l'héritage en CSS.

HTML (L'ancêtre de tous)
BODY (Parent)
HEADER (Enfant)
MAIN (Enfant)
<div class="parent">
    <!-- Ceci est un enfant du div -->
    <p>Je suis un paragraphe enfant.</p>
    
    <!-- Ceci est un autre enfant (frère du p) -->
    <span>Je suis un texte court.</span>
</div>

La Structure de Base

Tout fichier HTML commence par ces deux grandes parties :

1. Le HEAD (<head>)

C'est le "cerveau" de la page. Rien de ce qui est ici n'est affiché directement à l'utilisateur, mais c'est essentiel pour le navigateur.

  • Meta Charset : Pour gérer les accents (UTF-8).
  • Title : Le titre dans l'onglet du navigateur.
  • Liens : Connexions vers le CSS, les polices (fonts) ou les icônes.
<head>
    <!-- Encodage des caractères (obligatoire) -->
    <meta charset="UTF-8">
    
    <!-- Titre de l'onglet -->
    <title>Ma Super Page</title>
    
    <!-- Lier le fichier CSS -->
    <link rel="stylesheet" href="style.css">
    
    <!-- Lier une police Google -->
    <link href="..." rel="stylesheet">
                
    <!-- Lier le fichier JS -->
    <script scr="..." defer> </script>
</head>

2. Le BODY (<body>)

Le corps représente toute la page visible. Tout ce que l'utilisateur voit doit être entre les balises body.

On distingue deux types de balises dans le body :

A. Les Conteneurs (Structure)
  • <header> : En-tête du site (logo, menu).
  • <nav> : Liens de navigation.
  • <main> : Contenu principal de la page.
  • <footer> : Pied de page (copyright, contacts).
  • <section> : Pour découper le contenu par thématique.
  • <div> : Le conteneur "joker" générique. À utiliser si aucune balise sémantique ne correspond.
B. Le Contenu (Texte & Média)
  • <h1> à <h6> : Les titres (h1 est le plus important).
  • <p> : Les paragraphes de texte.
  • <img> : Les images (balise auto-fermante).
  • <a> : Les liens hypertextes (Anchor).

2. CSS : L'Habillage

Le CSS (Cascading Style Sheets) gère l'apparence. Il cible des éléments HTML pour leur appliquer des règles visuelles.

Comment cibler les éléments ? (Sélecteurs)

Il y a une hiérarchie de précision pour dire au navigateur "Je veux styliser CET élément" :

  • La Balise (Peu précis) : Cible TOUS les éléments de ce type.
    Ex: p { ... } stylise tous les paragraphes.
  • La Classe (Le standard) : Cible un groupe d'éléments spécifiques. On utilise un point .
    Ex: .bouton-bleu { ... }. À privilégier.
  • L'ID (Très précis) : Cible UN SEUL élément unique. On utilise un dièse #.
    Ex: #logo-principal { ... }.
/* Cible toutes les balises <p> */
p {
    color: grey;
}

/* Cible les éléments avec class="card" (Le plus courant) */
.card {
    background-color: white;
    padding: 20px;
}

/* Cible l'élément avec id="header-main" */
#header-main {
    position: fixed;
}

La Mise en Page (Layout)

C'est ici que la magie opère. Oubliez les float d'il y a 10 ans. Aujourd'hui, on utilise deux outils puissants.

1. Display: Flex (Flexbox)

Idéal pour les alignements en une dimension (une ligne OU une colonne). Parfait pour les menus, centrer des éléments, etc.

Exemple : display: flex; justify-content: space-between;

.menu {
    display: flex;           /* Active le mode Flex */
    justify-content: center; /* Aligne horizontalement */
    align-items: center;     /* Aligne verticalement */
    gap: 1rem;               /* Espace entre les enfants */
}

2. Display: Grid

Idéal pour les structures en deux dimensions (lignes ET colonnes). Parfait pour les grilles d'images ou la structure globale de la page.

Exemple : grid-template-columns: repeat(3, 1fr);
